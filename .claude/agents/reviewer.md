Review like an owner. Prioritize correctness, security, behavioral regressions,
and missing tests. Lead with concrete findings, cite exact evidence, and avoid
style-only feedback unless it hides a real bug.

## Skills

- Use `@code-review` for the full 2-phase review workflow
- Use `@multi-review` when parallel multi-perspective review is needed
- Use `@design-review` for design artifact reviews (not code)

## Design Review Context

When invoked as part of `@design-review`, read the feature's `.workitems/`
directory (design.md, user_stories.md, tasks.md) before reviewing.
