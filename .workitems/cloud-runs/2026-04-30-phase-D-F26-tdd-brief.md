### Mocking strategy

The `google-cloud-texttospeech` client is unit-tested by mocking the
`TextToSpeechClient.synthesize_speech` method using
`unittest.mock.patch`. Tests construct a synthetic
`SynthesizeSpeechResponse` with `audio_content` + `timepoints` lists,
patch the client, and assert F26's processing.

**Reference shape** for a mock response (per `texttospeech_v1beta1`):

```python
from unittest.mock import MagicMock
mock_response = MagicMock()
mock_response.audio_content = b"<mp3 bytes>"
mock_response.timepoints = [
    MagicMock(mark_name="b1-0", time_seconds=0.0),
    MagicMock(mark_name="b1-1", time_seconds=0.5),
]
```

### Mark truncation detection (T-04, post-review C2)

The adapter MUST set `voice_meta["tts_marks_truncated"] = True` when
the SSML's `<mark>` count differs from the response's timepoint count.
F30 already consumes this flag; verify the wire by calling F30 with
the F26-produced TTSResult in a smoke test.

```python
def _count_input_marks(ssml: str) -> int:
    # POC: simple substring count; ElementTree-based count for hardening
    return ssml.count("<mark ")

def _detect_truncation(input_ssml: str, response_timepoints) -> bool:
    return _count_input_marks(input_ssml) != len(response_timepoints)
```

### Drafts dir write (T-05)

Per design.md: write `drafts/<reading-plan-sha>/<block_id>.mp3` and
`<block_id>.audio.json`. Path format pinned by F35's consumer.

```python
from pathlib import Path

def write_drafts(
    *,
    reading_plan_sha: str,
    block_id: str,
    audio_bytes: bytes,
    timing_result,  # WordTimingResult-shape dict
) -> tuple[Path, Path]:
    drafts = Path("drafts") / reading_plan_sha
    drafts.mkdir(parents=True, exist_ok=True)
    mp3_path = drafts / f"{block_id}.mp3"
    json_path = drafts / f"{block_id}.audio.json"
    mp3_path.write_bytes(audio_bytes)
    json_path.write_text(json.dumps(timing_result, ensure_ascii=False, indent=2))
    return mp3_path, json_path
```

Tests use `tmp_path` fixture; assert files exist with correct content.
